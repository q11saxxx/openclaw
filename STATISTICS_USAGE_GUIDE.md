# 审计统计分析功能 - 使用指南

## 🚀 快速开始

### 1. 启动服务

```bash
# 终端1: 启动后端
cd c:\code\openclaw-finally\openclaw-final
uvicorn app.main:app --reload

# 终端2: 启动前端
cd c:\code\openclaw-finally\openclaw-final\frontend
npm run dev
```

### 2. 访问统计分析页面

打开浏览器访问: **http://localhost:3000/statistics**

或者从Dashboard点击"统计分析"卡片进入。

---

## 📊 功能详解

### 一、智能洞察面板

页面顶部显示4个智能洞察卡片，自动分析系统状态：

#### 洞察类型
- 🔵 **Info（蓝色）** - 一般信息提示
  - 示例: "过去7天进行了 X 次审计，系统使用频繁"
  
- 🟠 **Warning（橙色）** - 需要注意的警告
  - 示例: "过去30天 X/Y 个Skill存在高风险，建议加强审查"
  
- 🟢 **Success（绿色）** - 积极的改进信号
  - 示例: "近期高风险Skill数量下降，安全状况正在改善"
  
- 🔴 **Error（红色）** - 需要立即处理的问题
  - 示例: "检测到异常审计模式"

#### 交互效果
- 鼠标悬停时卡片轻微上浮
- 渐变背景色增强视觉层次
- 图标和文字清晰可读

---

### 二、风险等级分布饼图

**位置**: 左侧图表区域

**展示内容**:
- Critical（严重）- 红色扇区
- High（高危）- 橙色扇区
- Medium（中等）- 蓝色扇区
- Low（低危）- 绿色扇区

**使用方法**:
- 鼠标悬停查看具体数量和百分比
- 点击图例可隐藏/显示特定类别
- 环形设计美观且信息密度高

**数据更新**: 
- 根据顶部时间选择器自动刷新（7/30/90天）

---

### 三、审计频率柱状图

**位置**: 右侧图表区域

**展示内容**:
- X轴: 日期（月-日格式）
- Y轴: 审计次数
- 渐变色柱子（紫色到蓝色）

**使用方法**:
- 鼠标悬停查看具体日期的审计次数
- 观察审计活动的活跃度变化
- 识别审计高峰期

**视觉特点**:
- 渐变色填充增加现代感
- 柱子宽度适中，易于阅读
- 坐标轴标签旋转45度避免重叠

---

### 四、风险趋势分析折线图

**位置**: 页面中部大图表

**展示内容**:
- 四条堆叠面积曲线分别代表四个风险等级
- X轴: 时间序列
- Y轴: 各风险等级的数量

**颜色编码**:
- Critical: 红色 (#f56c6c)
- High: 橙色 (#e6a23c)
- Medium: 蓝色 (#409eff)
- Low: 绿色 (#67c23a)

**使用方法**:
- 鼠标悬停查看某一天的详细数据
- 点击图例过滤显示特定风险等级
- 观察长期趋势和周期性变化

**分析价值**:
- 识别风险等级的演变模式
- 发现潜在的安全问题集中爆发期
- 评估安全措施的有效性

---

### 五、Skill版本对比查询

**位置**: 页面底部

#### 查询步骤

1. **输入Skill名称**
   - 在文本框中输入要查询的Skill名称
   - 支持模糊匹配（需完整名称）
   - 可点击清除按钮重新输入

2. **点击"查询对比"**
   - 触发API请求获取历史数据
   - 显示Loading状态
   - 异步加载不阻塞页面

3. **查看结果**
   - 顶部显示审计记录总数
   - 趋势提示条显示风险变化方向
   - 表格展示详细审计历史

#### 结果表格字段

| 列名 | 说明 | 示例 |
|------|------|------|
| 版本 | Skill版本号 | 1.0.0 |
| 审计时间 | 审计执行时间 | 01-15 10:30 |
| 风险等级 | Critical/High/Medium/Low徽章 | 🔴 Critical |
| 置信度 | AI判断的可信度百分比 | 85% |
| 发现数量 | 检测到的风险项数量 | 5 |
| 操作 | 查看报告链接 | [查看报告] |

#### 趋势指示

- 📈 **improving（绿色）** - 风险状况正在改善
- ⚠️ **worsening（红色）** - 风险状况有所恶化
- ➡️ **stable（蓝色）** - 风险状况保持稳定
- ❓ **unknown（灰色）** - 数据不足，无法判断

- 一目了然的安全概览

## 🔧 高级技巧

### 1. 自定义时间范围

虽然UI提供7/30/90天快捷选项，但可以通过修改URL参数自定义天数：

```
http://localhost:3000/statistics?days=60
```

### 2. 导出数据

目前图表支持：
- 右键保存图片（ECharts原生功能）
- 未来可扩展CSV/Excel导出

### 3. 性能优化

对于大量数据（>1000条审计记录）：
- 建议使用90天而非更长时间范围
- 图表会自动优化渲染性能
- 考虑后端实现分页或聚合

## ❓ 常见问题

### Q1: 为什么我的Skill查不到历史记录？

**A**: 可能原因：
1. Skill名称拼写错误（区分大小写）
2. 该Skill从未被审计过
3. 审计报告文件损坏或丢失

**解决方案**:
- 检查`data/reports/`目录下的JSON文件
- 确认Skill名称与manifest.yaml中一致
- 重新运行审计生成新报告

---

### Q2: 图表显示空白怎么办？

**A**: 可能原因：
1. 没有审计数据
2. ECharts未正确加载
3. 浏览器兼容性问题

**解决方案**:
- 刷新页面重试
- 检查浏览器控制台是否有错误
- 确认已安装echarts依赖 (`npm list echarts`)
- 尝试Chrome/Firefox等现代浏览器

---

### Q3: 智能洞察为什么只显示一条？

**A**: 洞察生成逻辑：
- 审计频率低时只显示"系统正常运行"
- 只有满足条件时才显示警告或改进提示

**解决方案**:
- 进行更多审计以触发更多洞察
- 检查是否存在高风险Skill
- 这是正常现象，表示系统健康

---

### Q4: 趋势图中的数据为什么不平滑？

**A**: 原因：
- 实际审计活动不是均匀分布的
- 某些日期可能没有审计
- 反映了真实的审计模式

**解决方案**:
- 使用更长的时间范围观察整体趋势
- 关注总体走向而非单日波动
- 考虑实施定期审计计划
