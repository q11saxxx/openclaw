# 模块分工建议

本文件说明：把项目拆成适合 4 人团队并行推进的工作包。

## 角色建议
### 1. 组长 / 架构与后端主程
- 维护仓库规范、接口规范、数据库模型
- 完成 `backend/app/api`、`backend/app/core`、`backend/app/db`

### 2. 核心引擎负责人
- 负责 `backend/app/engine`、`rules/`
- 对接 Trivy、SBOM、漏洞聚合、自研规则

### 3. 前端与可视化负责人
- 负责 `frontend/src/pages`、`components`、`stores`
- 完成仪表盘、任务列表、报告页

### 4. 测试与文档负责人
- 负责 `tests/`、`docs/`、`examples/`
- 维护演示数据、竞赛演示脚本、使用手册

## 每周交付节奏
- 周一：同步需求与接口
- 周三：模块开发推进
- 周末：联调与测试回归
